## 1. Executive Summary

Brain Builders is an AI-powered daily practice companion designed specifically for Australian children preparing for OC (Opportunity Class) and Selective School tests. The app complements existing tutoring programs by providing structured daily practice, adaptive learning, and engaging gamification to keep children motivated.

This document captures requirements gathered directly from Praptini, a 10-year-old student in Grade 4 transitioning to Grade 5, who served as the primary user and designer for the application.

### Key Value Propositions
- Daily structured practice (30 minutes minimum)
- Adaptive difficulty based on age and ability
- Engaging reward system with collectible cards
- Parent visibility into child's progress
- Safe social features for trading with friends

## 4. Core Features

### 4.1 Daily Practice Sessions

| Feature | Specification |
|---------|--------------|
| Minimum Duration | 30 minutes per day |
| Subjects | Maths & English (Phase 1) |
| Question Types | Multiple choice, fill-in, problem solving |
| Adaptive Difficulty | Adjusts based on age and performance |
| Progress Tracking | Tracks correct/incorrect, time spent, patterns |

### 4.2 Adaptive Learning System

The app uses an adaptive algorithm to personalise the learning experience:

- Questions get **HARDER** when child answers correctly
- Questions get **EASIER** when child struggles
- Difficulty calibrated to age/grade level
- Identifies weak areas and targets them
- Remembers performance over weeks/months

### 4.3 Wrong Answer Handling

When a child gets an answer wrong, the app provides supportive feedback:

1. Play a gentle 'incorrect' sound (not harsh)
2. **Show a character from favourite shows to cheer them up**
3. Provide encouragement (growth mindset language)
4. Offer hints or explanation of the correct answer
5. Adjust difficulty if struggling repeatedly

## 5. Gamification System

### 5.1 Reward Tiers by Age

| Age Group | Reward System |
|-----------|---------------|
| **Under 11 years** | Collectible Cartoon Cards |
| **11 years and over** | Real Piggy Bank (parent-approved money) |

### 5.2 Collectible Card System (Under 11)

#### Card Characters
Cards feature characters inspired by the child's favourite shows:
- **Violet, Klaus & Sunny Baudelaire** (A Series of Unfortunate Events)
- **Carmen Sandiego**
- **Sam & Cat** characters

#### Card Rarity System

| Rarity Level | Visual Characteristics |
|--------------|----------------------|
| Common | Standard border, regular colours |
| Uncommon | Subtle shine, slightly special colours |
| Rare | Shiny/glittery border |
| Ultra Rare | Shiny border + special colours + character in cool pose |

#### Collection Book
- Shows all cards the child **HAS** collected (full colour)
- Shows cards **NOT YET** collected as grey shadows
- Creates a "gotta catch 'em all" motivation

### 5.3 Real Piggy Bank System (11+)
- Children earn real money for completing tasks
- Parents must approve via email before money is added
- Creates real-world motivation for learning
- Parent controls how much money is rewarded

## 6. User Experience Design

### 6.1 Mystery Card Pack Opening

When a child earns a card pack for completing daily tasks:

1. Card pack appears on screen like a wrapped present
2. Child taps to open
3. **RIPPING SOUND** plays (like opening a gift)
4. **CONFETTI** animation fills the screen
5. Card is revealed with suspense/excitement
6. If RARE card: extra special animation/sound

### 6.2 Daily Task Completion

When a child finishes their daily 30-minute session:

- **CHEERING SOUND** plays (celebration!)
- **CONFETTI** animation on screen
- Mascot celebrates with them
- Mystery card pack is awarded
- Progress is saved and reported to parents

### 6.3 Engagement Requirements

> ‚ö†Ô∏è **CRITICAL**: The child will **STOP** using the app if it's not engaging.

The app must maintain engagement through:
- Constant positive feedback and rewards
- Exciting sounds and animations
- Characters the child loves
- Collectibles that create anticipation
- Adaptive difficulty (not too hard, not too easy)
- Social features with friends

## 8. Social Features

### 8.1 Friend System (Praptini's Design)

Praptini designed a safe friend system with multiple approval layers:

1. Child asks friend for their username **IN REAL LIFE**
2. Child searches for friend's username in app
3. Child sends friend request
4. **FRIEND** must accept the request
5. **BOTH PARENTS** must approve the friendship
6. Only then can they interact in the app

### 8.2 Card Trading
- Children can trade cards with approved friends only
- Both children must agree to the trade
- Creates social engagement and motivation
- No strangers can interact with the child
